# ARIA - AI Responsive Interactive Assistant

## アプリケーション仕様書 v1.0

作成日: 2026年3月11日

---

## 1. アプリ概要

### 1.1 アプリ名

**ARIA（アリア）** — AI Responsive Interactive Assistant

名前の由来: 音楽用語の「アリア（独唱曲）」と、AIが"語りかける"イメージを掛け合わせたもの。音声認識機能を持つAIアシスタントとして、ユーザーに寄り添う存在であることを表現している。

### 1.2 コンセプト

デスクトップに常駐し、ユーザーの作業を支援するAIアシスタントアプリケーション。テキストチャットと画面キャプチャ解析を核とし、コンテキストに応じた的確な回答を提供する。アイコンや外観を自由にカスタマイズでき、自分だけのAIパートナーとして使える。音声入力は将来フェーズで対応予定。

### 1.3 ターゲットユーザー

- PC作業中に素早くAIに質問したいユーザー
- ハンズフリーで操作したいユーザー
- 画面上の情報をそのままAIに読み取らせたいユーザー

## 2. 機能要件

### 2.1 常駐・起動機能

| 機能 | 詳細 |
|------|------|
| システムトレイ常駐 | タスクバー通知領域にアイコンとして常駐 |
| ホットキー起動 | グローバルショートカット（デフォルト: `Ctrl+Shift+A`）でチャットウィンドウをトグル表示 |
| スタートアップ登録 | OS起動時に自動的にバックグラウンド起動（オプション） |
| ウィジェットモード | デスクトップ上に小型のフローティングウィジェットとして常時表示（オプション） |

### 2.2 チャット機能

| 機能 | 詳細 |
|------|------|
| テキスト入力 | チャットウィンドウからテキストで質問・指示を送信 |
| マークダウン表示 | AIの回答をリッチテキスト（マークダウン）で表示 |
| コード表示 | シンタックスハイライト付きコードブロック表示 |
| 会話履歴 | セッション内の会話をスクロール表示・過去の会話を保存・検索 |
| コンテキスト維持 | 会話の文脈を保持した連続的な対話 |
| クリップボード連携 | クリップボードの内容を自動取得して質問に添付可能 |

### 2.3 アイコン・外観カスタマイズ機能

| 機能 | 詳細 |
|------|------|
| アイコン画像の変更 | ユーザーが任意の画像ファイル（PNG/JPG/SVG/ICO）をアプリアイコンとして設定可能 |
| トレイアイコンの変更 | システムトレイに表示されるアイコンもカスタム画像に変更可能 |
| ウィジェットアバター | フローティングウィジェットおよびチャット内のAIアバター画像をカスタマイズ可能 |
| プリセットアイコン | いくつかのプリセットアイコンを同梱（デフォルト、ダーク、ミニマル等） |
| 画像リサイズ・クロップ | 設定画面内で画像の切り抜き・リサイズに対応 |
| テーマカラー | アイコンに合わせてUIのアクセントカラーを変更可能 |

### 2.4 画面キャプチャ・解析機能

| 機能 | 詳細 |
|------|------|
| スクリーンショット取得 | ホットキー（`Ctrl+Shift+S`）またはボタンで画面全体/選択範囲をキャプチャ |
| 範囲選択キャプチャ | マウスドラッグで任意の範囲を選択してキャプチャ |
| OCRテキスト抽出 | 画面画像からテキストを抽出 |
| 画像解析 | キャプチャした画面の内容をAIが視覚的に理解・解析 |
| アクティブウィンドウ取得 | 現在アクティブなウィンドウの情報を自動取得 |

### 2.5 AI応答機能

| 機能 | 詳細 |
|------|------|
| LLMバックエンド | Claude API（Anthropic）を使用 |
| マルチモーダル対応 | テキスト＋画像を組み合わせた入力に対応 |
| システムプロンプト | カスタマイズ可能なペルソナ・指示設定 |
| ストリーミング応答 | トークン単位でリアルタイム表示 |
| ツール実行 | ファイル操作、Web検索、計算などのツール連携（拡張可能） |

## 3. 非機能要件

### 3.1 パフォーマンス

- アイドル時のCPU使用率: 1%以下
- メモリ使用量: 200MB以下（アイドル時）、500MB以下（アクティブ時）
- 起動時間: チャットウィンドウ表示まで0.5秒以内（ホットキー起動時）
- カスタムアイコン読み込み: 1秒以内（画像リサイズ含む）

### 3.2 セキュリティ

- APIキーのセキュアな保存（OS標準のキーチェーン/資格情報マネージャー使用）
- 画面キャプチャデータのローカル処理（送信時のみAPI経由）
- 会話履歴の暗号化保存
- カスタムアイコン画像のローカル保存（外部送信なし）

### 3.3 対応環境

- OS: Windows 10/11（初期リリース）、macOS対応（Phase 2）、Linux対応（Phase 3）
- ディスプレイ: マルチモニター対応
- ネットワーク: インターネット接続必須（AI API通信用）

### 4.2 主要技術選定

| レイヤー | 技術 | 選定理由 |
|---------|------|---------|
| デスクトップフレームワーク | **Electron** | クロスプラットフォーム対応、Web技術活用、豊富なエコシステム |
| フロントエンド | **React + TypeScript** | コンポーネント指向、型安全性、開発効率 |
| UIライブラリ | **Tailwind CSS + shadcn/ui** | 軽量かつモダンなデザイン |
| AI API | **Claude API (Anthropic)** | 高精度な日本語対応、マルチモーダル対応、ストリーミング対応 |
| 音声認識 (STT) | **Whisper.cpp（ローカル）** | オフライン対応可能、高精度、低レイテンシ |
| ウェイクワード | **Porcupine (Picovoice)** | 低負荷なオンデバイス処理、カスタムウェイクワード対応 |
| 音声合成 (TTS) | **Web Speech API / VOICEVOX** | 日本語の自然な読み上げ |
| スクリーンキャプチャ | **Electron desktopCapturer** + **screenshot-desktop** | ネイティブAPI活用 |
| OCR | **Tesseract.js** | ローカル処理、日本語対応 |
| データ保存 | **SQLite (better-sqlite3)** | 軽量、サーバー不要、高速 |
| パッケージ管理 | **pnpm** | 高速、ディスク効率 |
| ビルド・配布 | **electron-builder** | インストーラー生成、自動アップデート対応 |
| 開発ツール | **Claude Code** | AIアシステッドコーディング |

## 6. 開発フェーズ

### Phase 1: MVP（最小実行可能製品）— 4週間

- Electronアプリの基盤構築
- システムトレイ常駐＋ホットキー起動
- テキストチャット機能（Claude API連携）
- ストリーミング応答表示
- 会話履歴の保存・表示
- 基本的な設定画面（APIキー設定、アイコンカスタマイズ）

### Phase 2: 画面キャプチャ — 2週間

- スクリーンショット取得（全画面/範囲選択）
- キャプチャ画像のClaude Vision API送信
- OCRテキスト抽出
- クリップボード画像の貼り付け対応

### Phase 3: UX強化・安定化 — 2週間

- フローティングウィジェットモード
- アイコン・テーマカラーのカスタマイズ機能
- マルチモニター対応
- パフォーマンス最適化
- 自動アップデート機能
- インストーラー作成・配布準備

### Phase 4: 音声機能（後回し）— 3週間

- Whisper.cppによる音声認識統合
- ウェイクワード検知（Porcupine）による「Hey ARIA」起動
- 音声合成（TTS）による応答読み上げ
- マイク入力のUIフィードバック

### Phase 5: 拡張機能（将来）

- プラグインシステム（ツール拡張）
- Web検索連携
- ファイル操作ツール
- macOS / Linux対応
- ローカルLLM対応（オフラインモード）

## 9. リスクと対策

| リスク | 影響度 | 対策 |
|--------|--------|------|
| API通信遅延 | 中 | ストリーミング応答、タイムアウト処理、リトライ機構 |
| メモリ使用量の増大 | 中 | 会話履歴の上限設定、画像データの適切な解放 |
| スクリーンキャプチャの権限問題 | 高 | OS権限の事前チェック、ユーザーガイド整備 |
| APIコストの増大 | 中 | トークン使用量のモニタリング、要約によるコンテキスト圧縮 |
| カスタムアイコンの表示崩れ | 低 | 画像バリデーション、自動リサイズ・クロップ処理 |

## 10. 成功指標（KPI）

- 起動からAI応答表示までの平均時間: 3秒以内
- 画面キャプチャからOCR完了までの時間: 2秒以内
- アイドル時のリソース使用量: CPU 1%以下、メモリ 200MB以下
- アイコンカスタマイズの反映: リアルタイム（設定即反映）
- ユーザー満足度（テスト）: 4.0/5.0 以上
